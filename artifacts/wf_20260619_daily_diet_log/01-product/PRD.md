## 1. 목표

사용자가 하루 3분 이내에 다이어트 관련 기록을 남기고, 최근 변화와 기록 지속 여부를 한눈에 확인하게 한다.

성공 기준:

- 오늘 기록을 3분 이내에 저장할 수 있다.
- 날짜별 기록을 다시 열어 수정하거나 삭제할 수 있다.
- 최근 7일 및 30일의 체중·섭취 열량·운동 추이를 볼 수 있다.
- 입력값 오류나 저장 실패가 사용자에게 명확히 표시된다.

## 2. 대상 사용자와 핵심 시나리오

대상은 체중 감량 또는 식습관 관리를 위해 개인 기록을 꾸준히 남기려는 사용자다.

핵심 흐름:

1. 사용자가 앱을 열면 오늘 날짜의 기록 화면이 보인다.
2. 체중과 식사 항목을 입력하고 필요하면 운동·물·컨디션·메모를 추가한다.
3. 저장 후 오늘의 섭취 합계와 목표 대비 상태를 확인한다.
4. 달력 또는 기록 목록에서 과거 기록을 조회하고 수정한다.
5. 통계 화면에서 최근 체중 및 열량 추이를 확인한다.

## 3. MVP 기능 범위

### 필수

- 일일 기록 생성·조회·수정·삭제
- 날짜 선택 및 날짜별 단일 로그
- 기본 정보: 체중(kg), 목표 체중(설정값)
- 식사 기록: 구분(아침/점심/저녁/간식), 음식명, 열량(kcal), 선택 메모
- 운동 기록: 운동명, 시간(분), 선택 소모 열량
- 물 섭취량(ml), 컨디션(1~5), 자유 메모
- 일일 섭취 열량 자동 합산
- 최근 기록 목록 또는 달력 표시
- 최근 7일/30일 체중 및 섭취 열량 추이
- 빈 상태, 입력 검증, 저장 오류 처리
- 데이터 내보내기(JSON) 및 가져오기(JSON): 로컬 데이터 유실 대비

### 후속 범위

- 계정, 로그인, 클라우드 동기화
- 음식 데이터베이스 검색 및 바코드 인식
- 영양소(탄수화물·단백질·지방) 상세 분석
- 사진 첨부
- 푸시 알림, 연속 기록 보상
- AI 식단 평가 및 코칭
- 웨어러블·헬스 플랫폼 연동

## 4. 화면 구성

- `오늘`: 날짜, 체중, 식사 목록, 운동, 물, 컨디션, 메모, 저장 상태
- `기록`: 날짜별 기록 존재 여부와 요약, 과거 기록 열기
- `통계`: 7일/30일 범위 전환, 체중선, 일별 섭취 열량, 기록 일수
- `설정`: 목표 체중, 일일 목표 열량, 데이터 내보내기/가져오기, 전체 데이터 삭제

모바일 하단 내비게이션을 기본으로 하고 데스크톱에서는 폭을 제한한 단일 컬럼 또는 보조 패널 구성을 사용한다.

## 5. 핵심 데이터 규칙

- 날짜는 사용자 로컬 시간대의 `YYYY-MM-DD`를 식별자로 사용한다.
- 한 날짜에는 하나의 일일 로그만 존재한다.
- 체중은 20~500kg, 열량은 항목당 0~10,000kcal, 운동 시간은 0~1,440분 범위로 검증한다.
- 합계 열량은 식사 항목에서 계산하며 별도 입력값으로 신뢰하지 않는다.
- 삭제와 전체 데이터 초기화에는 확인 절차가 필요하다.

## 6. 비기능 요구사항

- 모바일 360px 이상에서 주요 입력과 저장이 가능해야 한다.
- 주요 동작은 키보드로 수행 가능하고 입력 요소에 명시적 레이블이 있어야 한다.
- 개인 건강 기록은 MVP에서 브라우저 로컬에만 저장하고 외부 전송을 하지 않는다.
- 새로고침과 브라우저 재시작 후에도 기록이 유지되어야 한다.
- 최근 1년치 기록 기준으로 목록과 통계가 즉시 반응해야 한다.

## 7. 수용 기준

- 기록이 없는 오늘 화면에서 필수 필드를 입력해 저장하면 같은 날짜로 다시 열 때 값이 유지된다.
- 식사 항목의 추가·수정·삭제에 따라 일일 합계가 즉시 갱신된다.
- 잘못된 숫자나 범위를 입력하면 저장되지 않고 해당 필드에 오류가 표시된다.
- 날짜를 변경하면 해당 날짜의 로그가 열리며 다른 날짜 데이터와 섞이지 않는다.
- 통계 범위에 포함된 저장 기록만 집계되고 누락 날짜는 임의의 0kg으로 표현되지 않는다.
- JSON 내보내기 후 데이터를 초기화하고 다시 가져오면 동일한 기록이 복원된다.

## 8. 제품 승인 게이트

신규 제품 범위 확정은 명시적 승인이 필요하다. 특히 다음 가정을 구현 전에 확인한다.

- 로그인 없는 단일 사용자 로컬 저장 방식
- 기본 열량 중심 기록(상세 영양소 제외)
- 모바일 우선 웹 앱
