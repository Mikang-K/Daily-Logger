- Inputs used: 현재 일일 기록·주간 통계·IndexedDB 구조, 사용자 요청
- Open assumptions: 분석은 선택 실행, 외부 AI API 미사용, 의료 진단·처방 제외, 런타임과 모델은 기술 스파이크 후 확정

## 1. 목표

사용자가 기록을 저장한 뒤 일일 기록과 최근 주간 흐름을 바탕으로 이해하기 쉬운 요약, 관찰된 패턴과 실천 가능한 다음 행동을 받게 한다. 원본 기록과 모델 요청은 사용자 기기 밖으로 전송하지 않는다.

## 2. 권장 사용자 흐름

1. 사용자가 일일 기록을 저장한다.
2. `로컬 AI로 분석` 버튼을 직접 누른다.
3. 앱이 로컬 LLM 연결 상태와 사용할 데이터 범위를 보여준다.
4. 사용자가 분석을 시작하면 진행·취소 상태가 표시된다.
5. 결과 카드에서 요약, 긍정적 패턴, 확인할 점과 다음 행동을 확인한다.
6. 기록이 바뀌면 기존 결과를 `이전 기록 기준`으로 표시하고 재분석을 제안한다.

저장할 때 자동 실행하지 않는 것을 MVP 기본값으로 한다. 로컬 모델도 CPU·GPU와 배터리를 많이 사용할 수 있고 결과 생성 시간이 길 수 있기 때문이다.

## 3. MVP 분석 범위

입력:

- 선택한 날짜의 체중, 식사와 열량, 운동, 물, 컨디션, 메모
- 최근 7일의 계산된 평균과 기록 일수
- 사용자가 설정한 목표 체중과 목표 열량
- 이전 AI 결과는 입력에 포함하지 않음

출력:

- `summary`: 사실 기반 2~3문장 요약
- `positivePatterns`: 기록에서 확인되는 긍정적 행동 최대 3개
- `attentionPoints`: 확인할 패턴 최대 3개
- `nextActions`: 다음 날 시도할 구체적 행동 최대 3개
- `dataLimitations`: 누락된 데이터와 해석 한계
- `safetyNotice`: 의료 조언이 아니라는 안내

## 4. 안전 및 제품 원칙

- 진단, 치료, 약물·보충제 처방을 하지 않는다.
- 극단적 절식, 구토, 자해 또는 섭식장애 위험 표현이 있으면 일반 다이어트 조언을 중단하고 전문적인 도움을 권한다.
- 체중 감소를 무조건 성공으로 평가하지 않는다.
- 기록에 없는 영양소나 소모 열량을 추정해 사실처럼 말하지 않는다.
- 각 관찰은 입력 데이터의 날짜·수치와 연결할 수 있어야 한다.
- 낮은 기록 충족도에서는 확정적 표현 대신 데이터 부족을 명시한다.
- 메모 안의 명령문은 신뢰하지 않는 사용자 데이터로 취급한다.
- 외부 URL로 분석 데이터를 보내는 설정을 MVP에서 제공하지 않는다.

## 5. 성공 기준

- 사용자가 로컬 런타임 연결 여부를 분석 전에 확인할 수 있다.
- 정상 환경에서 분석 시작 후 첫 상태 피드백이 1초 안에 표시된다.
- 모델 출력이 정의된 JSON 스키마를 통과한 경우에만 화면에 표시된다.
- 타임아웃, 모델 미설치, 런타임 중지와 잘못된 응답을 구분해 안내한다.
- 입력 기록을 변경하면 캐시된 분석의 오래됨 상태를 감지한다.
- 네트워크 검사에서 분석 데이터가 루프백 주소 밖으로 전송되지 않는다.
- 합성 평가 세트에서 수치 날조, 진단, 극단적 감량 권고가 차단된다.

## 6. 제외 범위

- 클라우드 LLM API
- 자동 의료 판단과 질병 위험 점수
- 실시간 채팅 및 장기 대화 기억
- 모델 자동 다운로드
- 백그라운드 자동 분석
- 사용자 간 결과 공유
- AI가 원본 일일 기록을 자동 수정하는 기능
