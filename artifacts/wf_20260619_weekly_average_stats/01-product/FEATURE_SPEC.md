# 주간 평균 통계 기능 요구사항

- Workflow ID: `wf_20260619_weekly_average_stats`
- Stage: `product_planning`
- Producing agent: `default` (product planning 역할 수행)
- Source task ID: `task_weekly_product_001`
- Timestamp: `2026-06-19T00:00:00+09:00`
- Summary: 일별 기록을 월요일~일요일 단위로 묶어 평균과 전주 대비 변화를 제공하는 기능의 범위를 정의한다.
- Inputs used: 현재 `InsightsScreen`, `calculateRangeStatistics`, 기존 PRD와 테스트
- Open assumptions: 한국식 주간 인식에 맞춰 월요일 시작, 서버·계정 없이 사용자 로컬 날짜 기준

## 1. 목적

일별 체중 변동에 과도하게 반응하지 않고 주간 흐름을 확인할 수 있게 한다. 사용자는 이번 주 평균과 지난주 평균을 비교하고, 최근 여러 주의 변화 방향을 파악할 수 있어야 한다.

## 2. 사용자 시나리오

1. 사용자가 `통계` 화면을 연다.
2. 이번 주의 평균 체중, 일평균 섭취 열량, 일평균 운동 시간을 확인한다.
3. 각 지표가 지난주보다 얼마나 변했는지 확인한다.
4. 최근 8주의 평균 체중 추이를 확인한다.
5. 기록이 부족한 주는 평균값과 함께 기록 일수를 확인해 통계의 신뢰도를 판단한다.

## 3. MVP 범위

### 필수

- 월요일 00:00부터 일요일 23:59까지를 한 주로 계산
- 이번 주와 지난주의 평균 비교
- 최근 8주 주간 평균 목록 또는 차트
- 주간 평균 체중
- 기록한 날 기준 일평균 섭취 열량
- 기록한 날 기준 일평균 운동 시간
- 각 지표별 표본 수 표시
- 전주 대비 증감량과 방향 표시
- 데이터 부족 및 빈 상태 처리
- 도메인 단위 테스트와 통계 화면 컴포넌트 테스트

### 후속 범위

- 주간 평균 수분 섭취와 컨디션
- 목표 열량 대비 주간 달성률
- 사용자가 주 시작 요일을 선택하는 설정
- 월간·분기별 집계
- 주간 리포트 이미지 또는 PDF 공유

## 4. 통계 정의

| 지표 | 계산식 | 표본 |
| --- | --- | --- |
| 평균 체중 | 해당 주 체중 합계 ÷ 체중 기록 일수 | 체중이 있는 날짜만 포함 |
| 일평균 섭취 열량 | 해당 주 총 섭취 열량 ÷ 일일 로그 수 | 저장된 일일 로그 포함 |
| 일평균 운동 시간 | 해당 주 총 운동 시간 ÷ 일일 로그 수 | 저장된 일일 로그 포함 |
| 전주 대비 변화 | 이번 주 평균 − 지난주 평균 | 양쪽 주에 비교 가능한 값이 있을 때만 표시 |

평균 체중은 소수점 둘째 자리에서 반올림해 한 자리까지 표시한다. 열량과 운동 시간은 정수로 반올림한다. 계산 결과는 저장하지 않고 원본 일일 로그에서 파생한다.

## 5. 데이터 충족도 규칙

- `기록 4/7일`처럼 일일 로그가 존재하는 날 수를 표시한다.
- 평균 체중에는 `체중 3회`처럼 실제 체중 표본 수를 별도로 표시한다.
- 체중 기록이 1회뿐이면 평균 자체는 표시하되 추세 해석에 주의가 필요하다는 문구를 표시한다.
- 지난주 값이 없으면 증감률 대신 `비교할 지난주 기록이 없어요`를 표시한다.
- 기록이 없는 날짜를 0kg, 0kcal 또는 0분 표본으로 포함하지 않는다.
- 미래 날짜는 이번 주 평균에 포함하지 않는다.

## 6. 화면 요구사항

`통계` 화면에 다음 순서로 배치한다.

1. 기간 제목: `이번 주 · 6월 15일–6월 21일`
2. 핵심 카드: 평균 체중, 평균 섭취 열량, 평균 운동 시간
3. 각 카드: 현재 평균, 전주 대비 값, 표본 수
4. 최근 8주 평균 체중 차트
5. 기록이 부족할 때 안내하는 빈 상태

증감은 색상만으로 표현하지 않고 화살표, 부호와 텍스트를 함께 사용한다. 체중 감소를 무조건 성공으로 표현하지 않으며 중립적으로 `지난주보다 0.4kg 낮음`처럼 표시한다.

## 7. 수용 기준

- 같은 주의 로그가 입력 순서와 무관하게 하나의 주간 집계로 묶인다.
- 일요일과 다음 월요일은 서로 다른 주로 집계된다.
- 체중이 없는 로그는 평균 체중 분모에서 제외된다.
- 기록이 없는 날은 섭취 열량 및 운동 시간 평균의 분모에서 제외된다.
- 현재 주의 미래 날짜 로그는 통계에서 제외된다.
- 이번 주와 지난주 값이 모두 있을 때 정확한 증감량이 표시된다.
- 지난주 또는 이번 주 표본이 없으면 잘못된 0이나 `NaN`을 표시하지 않는다.
- 최근 8주가 시간순으로 표시되고 각 주의 표본 수를 확인할 수 있다.
- 360px 화면에서 카드와 차트가 가로 스크롤 없이 읽힌다.

## 8. 승인 게이트

구현 전에 다음 제품 결정을 승인한다.

- 월요일 시작 주간 정의
- 열량·운동 평균을 `기록한 날` 기준으로 계산
- 최근 8주 표시
- 체중 표본 1회도 평균은 표시하되 낮은 신뢰도를 안내
